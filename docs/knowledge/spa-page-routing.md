# SPA とページ分割の考え方

React で作る画面でも、すべてを 1 つの URL に閉じ込める必要はない。

このリポジトリでは、ブラウザの戻る / 進む、リロード、URL 共有で復元したい単位は Next.js のページとして扱う。ページ内の一時的な操作は React state とモーダルで扱う。

## ページとして扱うもの

- ホーム
- 取引先一覧
- 取引先詳細
- 取引先責任者一覧
- 取引先責任者詳細
- 活動詳細
- 連携
- ごみ箱

対応する URL は次の通り。

| 画面 | URL |
| --- | --- |
| ホーム | `/` |
| 取引先一覧 | `/accounts` |
| 取引先詳細 | `/accounts/<取引先ID>` |
| 取引先責任者一覧 | `/contacts` |
| 取引先責任者詳細 | `/contacts/<取引先責任者ID>` |
| 活動詳細 | `/activities` |
| 連携 | `/integration` |
| ごみ箱 | `/recycle-bin` |

一覧から詳細へ移動した場合や、取引先詳細から取引先責任者詳細へ移動した場合は、ブラウザ履歴に積む。

主要タブ、一覧のレコード名、詳細間の移動リンクは `href` に実 URL を持たせる。通常クリックはシェル内のクライアントサイドルーティングで扱い、別タブやリンクコピーでは通常リンクとして扱えるようにする。

## 同じページ内で扱うもの

- 新規作成
- 編集
- 削除確認
- 復元確認
- 入力候補やメニューの開閉
- 通知

これらは、ユーザーが現在の一覧や詳細の文脈で行う操作として扱う。URL を分けず、モーダルや component state で管理する。

## 判断基準

ページにする目安:

- ブラウザの戻る / 進むで戻りたい
- リロードして同じ画面を復元したい
- URL を共有したい
- ナビゲーション上の現在地として扱いたい

ページ内状態にする目安:

- 一覧や詳細の上に一時的に出る
- キャンセルすれば元の画面へ戻る
- URL 共有の対象にしない
- 入力途中の一時状態である

## このリポジトリでの実装方針

- URL の入口は `app/<path>/page.tsx` に置く。
- 共通の Playground 表示処理は `app/playground-page.tsx` に寄せる。
- 画面の現在地は URL から復元する。
- 表示の組み立ては `components/Playground.tsx` と `components/playground/PlaygroundWorkspace.tsx` で行う。
- ページ扱いのナビゲーションやレコードリンクは `href` に実 URL を持たせ、通常クリックではシェルを残したまま URL と表示を切り替える。
- 画面切り替え後のデータ取得は、表示中のページに必要な API に絞る。
- 編集、削除確認、作成、候補 popup などの一時操作だけを component state で扱う。

React state はページ内の状態に使い、ページそのものの現在地は URL に置く。
