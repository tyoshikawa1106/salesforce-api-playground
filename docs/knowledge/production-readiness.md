# 業務本番化の検討観点

## このリポジトリで確認できること

このリポジトリでは、外部 SaaS 連携アプリの基本的な構成と開発運用を確認できます。

| 領域 | 確認できること |
| --- | --- |
| 外部サービス連携 | Salesforce OAuth、Client Credentials Flow、jsforce による Salesforce API 呼び出し |
| セッション管理 | token をブラウザへ出さず、暗号化した HttpOnly Cookie に session を保持する構成 |
| セキュリティ基礎 | OAuth state 検証、Origin / Referer 検証、Salesforce record id 検証、秘密情報の混入防止 |
| 権限確認 | Salesforce 標準オブジェクトの `describe()` による操作前の権限確認 |
| 入力検証 | Account / Contact の許可フィールド、必須値、更新 payload、検索文字列の検証 |
| UI | SLDS ベースの接続、一覧、作成、編集、削除、検索、通知、環境ラベル表示 |
| 開発フロー | GitHub Flow、Issue / PR、Draft PR、CI pass 後の ready for review |
| 品質ゲート | lint、SLDS lint、typecheck、Vitest coverage、build、workflow YAML check |
| デプロイ運用 | GitHub `main` merge 後の Staging 自動デプロイ、Production promote を前提にした Heroku Pipeline 運用 |
| ドキュメント | README を入口にし、設計、セットアップ、セキュリティ、運用、ナレッジを `docs` に整理する構成 |

## 業務本番化するときの検討観点

業務本番化を考えるときは、機能が動くことだけでなく、利用者、権限、監査、運用、復旧、データ管理を設計対象に含めます。

| 観点 | 検討すること |
| --- | --- |
| 利用者と権限 | 想定利用者、管理者と一般ユーザーの権限差、Salesforce 側権限との分担 |
| 認可 | 画面、API、データ、操作ごとの許可条件、職務分掌、承認フロー |
| 監査 | 誰が、いつ、何を、どのデータに対して実行したかをどこへ保存するか |
| 可観測性 | structured log、request id、trace id、metric、alert、dashboard |
| セキュリティ | rate limit、CSRF、CORS、依存関係脆弱性、secret rotation、token 失効手順 |
| テスト | unit、integration、E2E、contract test、manual verification の分担 |
| データ管理 | アプリ DB の有無、履歴、復旧、再実行、整合性、バックアップ |
| 運用復旧 | rollback、障害切り分け、暫定対応、復旧手順、責任者、連絡経路 |
| リリース | Staging 確認、本番反映、承認、変更凍結、緊急修正 |
| コンプライアンス | 個人情報、ログ保存期間、監査証跡、アクセスレビュー、規程との整合 |

これらは、機能追加のついでに後から追加するより、業務要件が見えた時点で最初に境界を決める方が安全です。特に、監査、認可、データ保存、運用復旧は後から追加すると設計全体に影響しやすいため、早い段階で必要性を確認します。

## 判断の目安

検証段階で扱うか、業務本番化時の検討事項として外に置くかは、次の問いで判断します。

| 問い | 入れる判断 | 外に置く判断 |
| --- | --- | --- |
| 検証目的に直結するか | その仕組み自体を学びたい、または主要機能を理解するために必要 | 本番なら必要だが、今回の検証対象ではない |
| 小さく試せるか | 数ファイル、少数のテスト、短い docs で検証できる | 設計、運用、権限、データ保存まで広がる |
| 失敗時の影響は小さいか | ローカルや検証環境で壊れても復旧が容易 | 利用者、データ、外部サービス、運用体制に影響する |
| 継続運用できるか | CI や docs で維持できる | 監視、当番、棚卸し、定期レビューが必要 |
| 将来の再利用性があるか | 他プロジェクトにも使える構成や知見になる | 特定業務や組織ルールに強く依存する |

外に置く判断をしたものも、Issue や docs に「本番化時に検討する論点」として残すと、検証段階の軽さと将来の拡張性を両立できます。
